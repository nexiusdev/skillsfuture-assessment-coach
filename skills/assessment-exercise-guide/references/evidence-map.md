# Evidence Map

This exercise skill gathers evidence for LO1, LO2, LO3, LO4, LO5, and LO6. LO3 is coached as the final web-app build step after LO6 safety is passed.

| LO | Required Evidence | Exercise Artifact |
|---|---|---|
| LO1 | Workflow name, chosen F&B business type, pain points, dependencies, constraints, baseline frequency, average time per run, target ROI or success metric | `assessment-notes/LO1-workflow-selection.md` |
| LO2 | DUO teaching notes, Discover/Understand/Output learner evidence, CEO daily report requirements, priority definitions, `.codex-plugin/plugin.json`, `SKILL.md`, Normal/Messy/Edge test cases | `prompts/DUO-prompt.md`, `plugin/.codex-plugin/plugin.json`, `plugin/SKILL.md`, `assessment-notes/LO2-test-cases.md` |
| LO3 | Simple web app allowing branches to sign in with a simulated demo account, write or upload daily reports, keep historical records, retrieve records by branch/date/keyword, and show validation/error handling | `lo3-web-app/README.md`, `lo3-web-app/LO3-evidence.md`, app files inside `lo3-web-app/` |
| LO4 | Workflow diagram, trigger, branches, actions, logging, successful run log, failure alert proof | `automation/workflow-diagram.md`, `automation/run-log-success.md`, `automation/failure-alert-proof.md` |
| LO5 | Scope statement, goals, non-goals, tool limits, action limits, human approval gates | `governance/controlled-agent-scope.md` |
| LO6 | Safety checklist, redaction notes, no sensitive data confirmation, credential security, least privilege | `safety/safety-checklist.md`, `safety/redaction-notes.md`, `safety/credential-handling.md` |

## Pass-Ready Principle

Every claim in the final assessment portfolio should point to an artifact in the folder. If an artifact is missing, the assessment coach should mark the evidence as a gap rather than inventing it.
