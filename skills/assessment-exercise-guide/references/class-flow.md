# Instructor Class Flow

## Exercise Scenario

Learners act as F&B operations managers. They choose an F&B business type, define what the CEO needs in a daily operations report, and use Codex to build evidence for a controlled CEO daily reporting workflow.

This skill covers LO1, LO2, LO3, LO4, LO5, and LO6. LO3 is completed last as a simple branch daily-report web app build.

## Suggested Timing

| Segment | Duration | Output |
|---|---:|---|
| Briefing and folder scaffold | 15 min | Main exercise folder created |
| LO1 workflow selection | 30 min | Workflow selection note |
| LO2 DUO prompting practice | 45 min | DUO prompt evidence with learner answers, shared understanding, final prompt, and reflection |
| LO2 reusable skill/plugin | 45 min | Plugin scaffold and skill instructions |
| LO2 reliability tests | 45 min | Normal/Messy/Edge test notes |
| LO4 automation design | 45 min | Diagram, success log, failure alert proof |
| LO5 controlled agent scope | 30 min | Scope and approval gates |
| LO6 safety governance | 30 min | Safety checklist, credential handling, and redaction notes |
| LO3 web app build | 60 min | Simple branch daily-report web app and LO3 evidence |
| Final evidence check | 20 min | Folder ready for assessment coach |

## Instructor Notes

- Start by explaining the F&B CEO daily reporting scenario.
- Let learners choose their F&B business type so the exercise feels owned by them.
- Teach DUO as a shared AI-user context-building process, not only as a static prompt template.
- Keep all data dummy or anonymized.
- Let learners use simulated automation logs and alerts if live automation is not feasible, but require clear labeling.
- Do not accept evidence with secrets, real customer data, or private URLs.
- The final assessment report should be generated after the exercise by the assessment-submission-coach skill.
